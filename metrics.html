Realtime Metrics

The current runtime is optimized around conversational responsiveness rather than maximizing raw throughput. Performance tuning focused primarily on reducing prompt orchestration overhead, minimizing blocking operations, and allowing TTS synthesis to begin as early as possible during streaming generation.

Prompt Size
1.4k–4.3k
Runtime prompt size after retrieval and memory injection.

Earlier versions of the orchestration stack routinely exceeded 10k–13k character prompts, causing severe TTFT degradation and noticeable conversational dead air. The current system operates with substantially smaller prompts and asynchronously generated UI events, producing a significantly more realtime-feeling interaction loop.
